====== FINDOLOGIC Installationsanleitung für eigene Shopsysteme ======

Im folgenden erfahren Sie, wie Sie in wenigen Schritten FINDOLOGIC in ihr Shopsystem einbinden.
Die Komponenten die für die FINDOLOGIC Suche angepasst / entwickelt werden müssen betreffen:
  - [[#Export|Den Export der Artikeldaten]]
  - [[#Request|Die Erstellung der Suchanfrage]]
  - [[#Response|Die Verarbeitung der Suchergebnisse]]
  - [[#Smart_suggest|Die Einbindung der FINDOLOGIC-Smart Suggest(optional)]]
  - [[#Navigation|Die Einbindung der FINDOLOGIC Navigation (optional)]]
  - [[#Checkliste|Überprüfung der Einbindung anhand der FINDOLOGIC Zertifzierungskriterien (optional)]]

===== 1. Export =====
Um FINDOLOGIC zu ermöglichen die Artikeldaten zu indizieren ist zunächst ein Export der Artikeldaten notwendig.

Dafür sind folgende Schritte notwendig:
=== 1.1 Schnittstelle ===
  * Erstellen Sie eine Schnittstelle die von außen aufrufbar ist um den Export zu starten
=== 1.2 Export ===
  * Exportieren Sie Ihre Artikeldaten wie Artikel-ID, Beschreibung, Preis, Attribute, etc. in eine von außen erreichbare CSV-Datei oder per XML Export

==== Dokumentation ====

===== 2. Request =====
Um eine Suchanfrage an FINDOLOGIC zu senden, muss ein entsprechender HTTP Request an FINDOLOGIC gesendet werden.\\
Dieser entspricht einem einfachen definierten Aufbau mit vorgegebenen Parametern.

Dafür sind folgende Schritte notwendig:

=== 2.1 Zusammensetzung der Request-URL ===

  * Die Request-URL setzt sich aus dem vorgegebenen Pfad zum Service und den entsprechenden Suchparametern zusammen.
  * Die Request Parameter übermitteln FINDOLOGIC die genaue Suchanfrage.

== 2.1.1 Übermittlung der Suchbegriffe ==

  * Die Suchbegriffe werden FINDOLOGIC mit dem Parameter ''query'' übermittelt

== 2.1.2 Übermittlung der Attribute ==

  * Die Suchergebnisse können anhand von Attributen, Kategorie, Hersteller oder Preis verfeinert werden.
  * Anhand eines Parameters ''attrib'' werden die jeweiligen Einschränkungen an FINDOLOGIC übermittelt.

== 2.1.3 Übermittlung der Sortierung ==

  * Per default werden die Suchergebnisse von FINDOLOGIC nach der optimalen Relevanz sortiert.
  * Mit dem Parameter ''order'' können Sortierwünsche der Nutzer entsprechend angegeben werden.

== 2.1.4 Übermittlung des Shopkeys ==

  * Ein von FINDOLOGIC bereitgestellter Schlüssel definiert den zu durchsuchenden Service (Shop).
  * Dieser wird mittels des Parameters ''shopkey'' übermittelt.

== 2.1.5 Übermittlung weiterer Parameter ==

  * Zusätzlich sind Parameter zur Bestimmung der Anzahl der Ergebnisse, dem Offset oder der anfragenden User-IP zu übermitteln.

=== 2.2 Erstellung eines Fallback-Mechanismus ===

  * Sollte die FINDOLOGIC Suche nicht erreichbar sein, muss die herkömmliche Shopsuche die Suchanfrage verarbeiten.
  * Hierzu ist lediglich über einen definierten //Alivetest// (siehe [[fl:integration:request#fallback-mechanismus|Dokumentation]]) die Erreichbarkeit vorab zu prüfen und die Suchanfrage ggf. an die herkömmliche Shopsuche weiterzuleiten.

===== 3. Response =====
Die Rückgabe zu einer Suchanfrage kann von FINDOLOGIC wahlweise im XML oder HTML Format (gestützt durch Smarty-Templates) erfolgen.\\
Für die Implementation neuer Module empfehlen wir die Verwendung des XML Formats.

==== 3.a XML-Rückgabe ====
Die XML-Rückgabe besteht aus Informationen zur Suchanfrage, den Suchergebnissen, möglichen Filterwerten und Informationen zu speziellen Ausgabeformaten (Landingpage, Promotion).

=== 3.a.1 Verarbeitung der Suchergebnisse ===

  * Die Suchergebnisse werden als Produkt-IDs übergeben. Für die Darstellung werden gewöhnlicherweise bestehende Templates verwendet um die entsprechenden Produkte auszugeben.

=== 3.a.2 Verarbeitung der Filterwerte ===

  * Um dem Benutzer Filtermöglichkeiten anzubieten, müssen die zurückgegeben Filter und Filterwerte in eine HTML Darstelllung übertragen (gerendert) werden.

=== 3.a.3 Unterstützung zusätzlicher Features (spezielle Ausgabeformate) ===

  * Für Landingpages muss lediglich eine Weiterleitung zu einer übergebenen URL erfolgen.
  * Für Promotions muss das übergebene Bild mit Link über den Suchergebnissen (oder an einer beliebigen geeigneten Stelle) ausgegeben werden.

==== 3.b HTML-Rückgabe ====
Bei der Verwendung der HTML Rückgabe kommen Smarty Templates zum Einsatz.\\
Für die Anpassung der Templates bietet FINDLOGIC seinen Kunden einen Templatemanager im FINDOLOGIC Kundenaccount an.\\
Alle angebotenen Smarty-Variablen finden Sie in unten angeführter Dokumentation.\\
Für unsere Templates bieten wir zusätzlich eine Reihe an Plugins an, welche die Gestaltung der Templates für Sie sehr einfach machen.\\

=== 3.b.1 Ausgabe der Suchergebnisse ===

  * Die Ausgabe der Artikel kann über Smarty Templates im FINDOLOGIC Kundenaccount angepasst werden.
  * FINDOLOGIC empfiehlt allerdings, nur die IDs über die Templates ausgeben zu lassen und die Artikellistings mithilfe bestehender Shop-Templates darzustellen.

=== 3.b.2 Ausgabe der Filter ===

  * Die Ausgabe der Filter kann je nach Wunsch in den Templates angepasst werden

=== 3.b.3 Unterstützung zusätzlicher Features (spezielle Ausgabeformate) ===

  * Auch Landingpages und Promotions können direkt über die Smarty Templates im FINDOLOGIC Kundenaccount angepasst werden.

===== 4. Smart Suggest =====
Natürlich ist auch eine Smart Suggest teil des Funktionsumfangs von FINDOLOGIC.\\
Die Smart Suggest wird vollständig von FINDOLOGIC bereitgestellt. Zur Einbindung in den Shop sind nur wenige Schritte notwendig.\\
Auf Wunsch kann das Verhalten der Smart Suggest auch manuell angepasst werden.

Dafür sind folgende Schritte notwendig:

=== 4.1 Einbindung des Script-Tags ===

  * Für die Einbindung der Smart Suggest muss lediglich ein von FINDOLOGIC bereitgestellter Script-Tag in die Shopseite eingbunden werden.

=== 4.2 Hochladen des Smart Suggest-Proxys ===

  * Zusätzlich laden Sie eine vorgefertigte PHP-Datei in Ihr Shopverzeichnis hoch.

=== 4.3 Einstellungen ===

  * Sie können Einstellungen der Smart Suggest im FINDOLOGIC Kundenaccount durchführen

=== 4.4 Implementation ===

  * Es ist **keine** weitere **Implementation** für die Smart Suggest notwendig!

==== Dokumentation ====

Die genaue Dokumentation für die FINDOLOGIC-Smart Suggest finden Sie unter folgenden Links:

**Hier geht's zur [[fl:integration:smart_suggest|Einbindung der Smart Suggest ]]**\\

===== 5. Navigation =====
Die FINDOLOGIC-Navigation ermöglicht, dass der Kunde an beliebigen Stellen im Shop intuitiv den Artikel findet, den er wünscht - ganz unabhängig von der Benutzung der Suchfunktion!

Dafür sind folgende Schritte notwendig:

=== 5.1 Kategorie an FINDOLOGIC übermitteln ===

  * Bei der Navigation eines Kunden in eine Kategorie, wird eine Suchanfrage an FINDOLOGIC gesendet.

=== 5.2 Filter und Suchergebnisse darstellen ===

  * Die Verarbeitung der Rückmeldung erfolgt nach dem in [[#Response|Punkt 3]] beschriebenen Vorgehen.
