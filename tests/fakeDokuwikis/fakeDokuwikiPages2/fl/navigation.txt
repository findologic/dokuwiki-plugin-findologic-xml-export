====== FINDOLOGIC-Navigation ======

Die FINDOLOGIC-Navigation (ehem. FINDOLOGIC-Selector) ermöglicht, dass der Kunde an beliebigen Stellen im Shop intuitiv den Artikel findet, den er wünscht - ganz unabhängig von der Benutzung der Suchfunktion!

Dazu denkt die FINDOLOGIC-Navigation aktiv mit dem Käufer mit und schlägt entlang der jeweiligen Auswahl sinnvolle Verfeinerungsmöglichkeiten und passende Artikel vor. Die Vorschläge orientieren sich nach den Artikelmerkmalen des jeweiligen Shops. Wählt eine Kundin z.B. in einem Modeshop die Kategorie Damenschuhe an, werden ihr als weitere Filter z.B. "Art des Schuhs", "Schuhgröße", "Marke" und "Farbe" angeboten, innerhalb der sie dann ihren Wünschen entsprechend bequem auswählen kann. So finden die Kunden noch treffsicherer ihren Wunschartikel!

===== Integration =====

<note tip>Dieser Abschnitt behandelt die Integration der FINDOLOGIC-Navigation in noch nicht unterstützte Shop-Software aus technischer Sicht. Um die FINDOLOGIC-Navigation in Ihr bereits unterstütztes System zu integrieren, beachten Sie bitte die Dokumentation zur Integration von FINDOLOGIC in Ihrem Shop-System.</note>

Der Aufruf zur Navigation erfolgt wie gewohnt über die FINDOLOGIC-Schnittstelle in den zwei Varianten HTML und XML:

  * HTML: <code>http://<ServiceURL>/selector.php?shopkey=<SHOPKEY></code>
  * XML: <code>http://<ServiceURL>/ps/xml_2.0/selector.php?shopkey=<SHOPKEY></code>

Für die zur Verfügung stehenden Request-Parameter beachten Sie bitte die [[https://secure.findologic.com/dokuwiki/doku.php?id=fl:search_request_2_0|Dokumentation der Such-Parameter]]. Zusätzlich können Filter vorausgewählt werden, diese lassen sich vom Kunden nicht abwählen. So kann die Navigation direkt in einer gewünschten Kategorie (über die Kategorie-Navigation im Shops) oder für einen Hersteller starten. Ersetzen Sie dafür ''attrib'' durch ''selected'', ansonsten funktioniert dieser Parameter genau wie ''attrib''.

==== Ablauf ====

=== Abfrage der Filter ===

Im ersten Schritt werden die zur Verfügung stehenden Filter abgefragt, die vorgewählte Kategorie wird per ''selected''-Parameter übergeben. 

<note important>An dieser Stelle werden die Ergebnisse vom Shop, nicht von FINDOLOGIC generiert, da FINDOLOGIC hier noch keinen Mehrwert bietet; daher Abfrage mit ''count=0''</note>

Beispiel für einen Aufruf in der Kategorie "Mode": <code>http://<Service-URL>/selector.php?shopkey=ABCD&selected[cat][]=Mode&count=0</code>

Der Aufbau für Requests an FINDOLOGIC ist [[fl:integration:request|hier]] genau dokumentiert. Bitte achten Sie darauf bei jedem Aufruf ebenfalls die FINDOLOGIC Pflichtparamter mit anzugeben.

<note important>Achten Sie insbesondere darauf, die IP des anfragenden Endkunden mitzugeben, damit Anfragen von Bots erkannt werden können</note>

=== Ausgabe der Filter ===

Die Filter werden im Shop an entsprechender Stelle ausgegeben. Die Ausgabe wird im Modul im Shop realisiert.

Der Kunde kann nun einen FINDOLOGIC-Filter auswählen, wodurch im nächsten Schritt auch die Ergebnisse von FINDOLOGIC geholt werden.

=== Abfrage der Ergebnisse ===

Der Kunde hat einen Filter ausgewählt. Dieser wird nun an FINDOLOGIC übergeben

Beispiel für vorgewählte Kategorie "Mode" sowie Attribute "Farbe=blau": <code>http://<Service-URL>/selector.php?shopkey=ABCD&selected[cat][]=Mode&attrib[Farbe][]=blau</code>

==== Suchmaschinen ausschliessen ====
Dies kann nicht über die robots.txt realisiert werden, da ansonsten immer noch der erste Aufruf des Crawlers das FINDOLOGIC-Service aufruft. Daher muss abgefragt werden, ob ein Aufruf durch einen Crawler erfolgt und in diesem Fall die Auslieferung der Filter unterbunden werden.
